# Hevolve API - Endpoint Summary Reference

**API Base URL:** `http://aws_hevolve.hertzai.com:6006`

---

## Quick Endpoint Reference

### Posts Management
| Endpoint | Method | Purpose | Auth | Params | Response |
|----------|--------|---------|------|--------|----------|
| `/create_post` | POST | Create new post | - | body: CreatePost | 200 OK |
| `/get-all-post` | POST | Get all posts | - | none | 200 OK |
| `/get-posts` | POST | Get posts (paginated) | - | page_number, num_rows, user_id | 200 OK, 422 |
| `/delete_post/{postid}` | POST | Delete post by ID | - | path: postid, query: reason | 200 OK, 422 |

### Comments Management
| Endpoint | Method | Purpose | Auth | Params | Response |
|----------|--------|---------|------|--------|----------|
| `/Comment` | POST | Create comment on post | - | body: Command | 200 CommandBase, 422 |
| `/comment_bypost` | GET | Get comments for post | - | query: post_id, user_id | 200 OK, 422 |
| `/delete_comment` | DELETE | Delete comment | - | query: comment_id, reason | 200 OK, 422 |

### Likes Management
| Endpoint | Method | Purpose | Auth | Params | Response |
|----------|--------|---------|------|--------|----------|
| `/Like` | POST | Like post/comment | - | body: Like | 200 OK, 422 |
| `/like_bypost` | GET | Get likes for post | - | query: post_id | 200 OK, 422 |

### Friends/Users Management
| Endpoint | Method | Purpose | Auth | Params | Response |
|----------|--------|---------|------|--------|----------|
| `/add_friend` | POST | Add friend | - | body: add_friend | 200 OK, 422 |
| `/allusers` | GET | List all users | - | query: skip, limit | 200 OK, 422 |
| `/action_by_user_id` | GET | Get user actions | - | query: user_id | 200 OK, 422 |

### Blocking Management
| Endpoint | Method | Purpose | Auth | Params | Response |
|----------|--------|---------|------|--------|----------|
| `/block_user` | POST | Block a user | - | body: block_user | 200 OK, 422 |

## Request/Response Schema Quick Reference

### CreatePost Request
```javascript
{
  "user_id": integer,           // Required
  "resource_url": string,       // Optional - URL to media
  "content_type": string,       // Optional - "image", "video", "text", etc.
  "caption": string,            // Optional - Post text
  "repost": boolean,            // Optional, default: false
  "parent_post_id": integer     // Optional - ID of original post if reposting
}
```

### Command Request (Comment)
```javascript
{
  "post_id": integer,           // Required
  "user_id": integer,           // Required
  "text": string,               // Required
  "parent_comment_id": integer  // Optional - For nested comments
}
```

### CommandBase Response (Comment)
```javascript
{
  "post_id": integer,
  "user_id": integer,
  "text": string,
  "parent_comment_id": integer,
  "comment_id": integer         // Generated by server
}
```

### Like Request
```javascript
{
  "activity_id": integer,       // Required - Post or comment ID
  "user_id": integer            // Required
}
```

### LikeBase Response
```javascript
{
  "activity_id": integer,
  "user_id": integer,
  "like_id": integer            // Generated by server
}
```

### add_friend Request
```javascript
{
  "user_id": integer,           // Required - Requester
  "friend_user_id": integer     // Required - Target user
}
```

### block_user Request
```javascript
{
  "user_id": integer,           // Required - Blocker
  "block_user_id": integer,     // Required - Blocked user
  "type_of_activity": string,   // Required - "posts", "comments", "messages"
  "scope": string               // Required - "global", "posts_only", "messages_only"
}
```

## Common Query Parameter Patterns

### Pagination Parameters
| Parameter | Type | Default | Usage |
|-----------|------|---------|-------|
| `page_number` | integer | 1 | For paginated endpoints (get-posts) |
| `num_rows` | integer | 10 | Rows per page |
| `skip` | integer | 0 | Offset for cursor-based pagination (allusers) |
| `limit` | integer | 100 | Max results to return |

### Filtering Parameters
| Parameter | Type | Endpoint | Usage |
|-----------|------|----------|-------|
| `user_id` | integer | get-posts, comment_bypost, action_by_user_id | Filter by user |
| `post_id` | integer | comment_bypost, like_bypost | Get items for specific post |
| `comment_id` | integer | delete_comment | Specific comment to delete |
| `postid` | integer | delete_post/{postid} | Specific post to delete |

### Optional Parameters
| Parameter | Type | Endpoint | Usage |
|-----------|------|----------|-------|
| `reason` | string | delete_post, delete_comment | Reason for deletion |

## Common Workflows

### Create & Comment on Post
```
1. POST /create_post             → Returns post ID
2. POST /Comment                 → Add comment to post
3. GET /comment_bypost           → View all comments
4. DELETE /delete_comment        → Remove comment if needed
```

### Like & Engagement
```
1. POST /Like                    → Like post/comment
2. GET /like_bypost              → See all likes on post
3. GET /comment_bypost           → Get engagement metrics
```

### Social Network
```
1. GET /allusers                 → Browse users
2. POST /add_friend              → Connect with user
3. POST /block_user              → Block if needed
4. GET /action_by_user_id        → See user's activity
```

### Feed Management
```
1. POST /get-posts?page=1        → Load initial feed
2. POST /get-posts?page=2        → Load more (pagination)
3. Filter by user_id             → View user's posts only
4. POST /delete_post/{id}        → Remove own post
```

## API Capabilities Summary

### What You Can Do

**Content Creation:**
- Create text, image, and video posts
- Support nested reposts
- Add comments and threaded replies

**Content Discovery:**
- Browse community feed (paginated)
- Filter posts by user
- View all users with pagination

**Engagement:**
- Like posts and comments
- Leave nested comments/replies
- Track activity by user

**Social Features:**
- Add friends/connections
- Block users with granular control
- Get user activity history

### Limitations to Note

- No built-in search (would need to filter client-side or use separate search endpoint)
- No direct messaging API shown (though blocking supports "messages")
- No media upload endpoint shown (resource_url is external)
- No notifications/feed algorithm shown
- No follower/following lists shown (just friends)
- No post editing (delete and recreate)
- No comment editing (delete and recreate)
- No analytics/metrics endpoints shown

## Best Practices

### 1. Error Handling
```javascript
try {
  const response = await fetch(endpoint, options);
  if (response.status === 422) {
    // Handle validation error
    const error = await response.json();
    console.error('Validation failed:', error.detail);
  } else if (response.ok) {
    // Handle success
    const data = await response.json();
  }
} catch (error) {
  // Handle network error
  console.error('Network error:', error);
}
```

### 2. Pagination Strategy
```javascript
// Always paginate large requests
const pageSize = 10;
let pageNumber = 1;
let hasMore = true;

while (hasMore) {
  const response = await fetch(
    `/get-posts?page_number=${pageNumber}&num_rows=${pageSize}`,
    { method: 'POST' }
  );
  const posts = await response.json();

  if (posts.length < pageSize) {
    hasMore = false;
  }
  pageNumber++;
  // Process posts...
}
```

### 3. Validation Before Submitting
```javascript
function validatePost(post) {
  if (!post.user_id || typeof post.user_id !== 'number') {
    throw new Error('Invalid user_id');
  }
  if (post.caption && post.caption.length > 5000) {
    throw new Error('Caption too long');
  }
  return true;
}
```

### 4. Rate Limiting Considerations
```javascript
// Implement delays for bulk operations
async function bulkLikePosts(postIds, userId) {
  for (const postId of postIds) {
    await likePost(userId, postId);
    await sleep(100); // 100ms delay between requests
  }
}
```

### 5. Local Caching
```javascript
const postCache = new Map();

async function getPost(postId) {
  if (postCache.has(postId)) {
    return postCache.get(postId);
  }

  // Fetch from API if not cached
  const post = await fetchPost(postId);
  postCache.set(postId, post);
  return post;
}
```
